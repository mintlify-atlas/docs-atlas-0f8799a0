This guide walks you through your first batch operation using the **Compress** module, which is the simplest way to see Yagua in action.

<Note>
  Make sure you have Yagua installed before continuing. See the [installation guide](/installation) if you haven't set it up yet.
</Note>

## Your first batch operation

<Steps>
  <Step title="Open the app">
    Launch Yagua from your desktop shortcut (Windows), by running the AppImage (Linux), or with `python main.py` if you're running from source.

    The main window opens with the sidebar on the left and the module workspace on the right.
  </Step>

  <Step title="Select a module from the sidebar">
    In the left sidebar, click **Compress**. The sidebar lists all available modules — each one corresponds to a specific processing task.

    The **Compress** module workspace loads on the right side of the window.

    <Tip>
      You can control which modules appear in the sidebar from **Settings**. If a module you expect isn't visible, check that it's enabled there.
    </Tip>
  </Step>

  <Step title="Load your images">
    Click the **Add images** button (or drag and drop files into the drop zone) to load the images you want to process.

    You can load up to **100 images** per batch in the Compress module.

    <Note>
      The background removal module has a lower limit of **10 images** per batch due to the processing demands of the AI model.
    </Note>
  </Step>

  <Step title="Configure compression settings">
    Adjust the quality slider to your target quality level. A lower value produces smaller files; a higher value preserves more detail.

    You can also choose the output format — JPEG, PNG, WEBP, AVIF, HEIC, and others are supported. Leave the format unchanged to keep the original format of each file.
  </Step>

  <Step title="Run the batch">
    Click **Process** to start the batch operation. A progress indicator shows the status of each image as it is compressed.

    Yagua processes all images locally on your machine — nothing is uploaded to any server.
  </Step>

  <Step title="Save the output files">
    When processing is complete, click **Save** or **Open output folder** to access the results. Output files are saved to the folder you selected, keeping your originals intact.
  </Step>
</Steps>
